# Absensi Sekolahku: Project Architecture Blueprint 🚀

Dokumen ini berisi panduan teknis, arsitektur, dan tech stack yang digunakan dalam project "Absensi Sekolahku" sebagai standar untuk pengembangan project skala enterprise lainnya.

## 1. Core Technology Stack

### Frontend (Modern React Ecosystem)
- **Framework**: [React 19](https://react.dev/) (Vite sebagai build tool).
- **Styling**: [Tailwind CSS 4](https://tailwindcss.com/) (Modern, utility-first, performant).
- **State Management**: [Zustand](https://github.com/pmndrs/zustand) (Ringan, tanpa boilerplate redux).
- **Data Fetching**: [TanStack Query v5](https://tanstack.com/query) (Server-state management, caching, auto-sync).
- **Forms & Validation**: [Formik](https://formik.org/) + [Yup](https://github.com/jquense/yup).
- **UI Components**: [Radix UI](https://www.radix-ui.com/) (Primitives) + [Lucide React](https://lucide.dev/) (Icons).
- **HTTP Client**: [Axios](https://axios-http.com/) (Custom instance dengan Refresh Token Interceptor).

### Backend (Robust Node.js)
- **Runtime**: Node.js (Version 22 LTS).
- **Framework**: [Express.js](https://expressjs.com/).
- **ORM**: [Sequelize](https://sequelize.org/) (MySQL / MariaDB).
- **Authentication**: [JWT](https://jwt.io/) (Access Token) + [HttpOnly Cookies](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies) (Refresh Token).
- **Logging**: Custom JSON Logger (Audit Trail).

## 2. Frontend Architecture (Feature-Driven Design)

Struktur folder di `client/src` dibagi berdasarkan **Feature**, bukan berdasarkan jenis file (seperti `components/` global yang menumpuk).

### Folder Structure:
```text
src/
├── features/               # Modul-modul utama aplikasi
│   ├── auth/               # Modul Login, Logout, Refresh
│   ├── teacher-attendance/ # Fitur spesifik absen guru
│   ├── master-data/        # Pengelolaan Guru, Kelas, Jadwal
│   └── [feature-name]/
│       ├── components/     # UI khusus fitur tersebut
│       ├── hooks/          # Custom hooks (e.g. useAttendanceData)
│       ├── services/       # API call khusus fitur
│       ├── interfaces/     # TypeScript definitions
│       └── pages/          # Entry point halaman fitur
├── shared/                 # Kode yang dipakai di banyak fitur
│   ├── components/         # UI Primitives (Button, Modal, Input)
│   ├── lib/                # Config Axios, Utils
│   ├── store/              # Global Zustand stores
│   └── routes/             # Routing configuration
└── assets/                 # Images, Fonts, Styles
```

## 3. Backend Architecture (Modular Pattern)

Backend dipisahkan berdasarkan tanggung jawab (Separation of Concerns) agar mudah di-maintain dan di-test.

### Folder Structure:
```text
server/
├── controllers/    # Logika bisnis utama (Request/Response handling)
├── routes/         # Definisi endpoint API
├── models/         # Skema database (Sequelize Models)
├── middleware/     # Auth check, error handling, file upload
├── utils/          # Helper functions
├── uploads/        # Static files storage
└── index.js        # Entry point aplikasi
```

## 4. Key Implementation Patterns

### A. Seamless Authentication (Best Practice)
- **Access Token**: Disimpan di memory (State), umur pendek (15m-1h).
- **Refresh Token**: Disimpan di **HttpOnly Cookie**, umur panjang (7d).
- **Axios Interceptor**: 
    1. Intercept `401 Unauthorized`.
    2. Jalankan background request ke `/auth/refresh`.
    3. Update Access Token di state.
    4. **Retry** request awal yang gagal tadi.

### B. Database Snapshotting (History Integrity)
- Data transaksi (seperti `Activity`) tidak hanya menyimpan ID, tapi melakukan **Snapshotted Data** (menyimpan string Nama Guru, Nama Kelas, dll pada saat transaksi dibuat).
- Hal ini menjamin laporan historis tetap valid meskipun data Master (Guru/Kelas) dihapus atau diubah di masa depan.

### C. Strong Typing
- Wajib menggunakan TypeScript (atau JSDoc jika terpaksa).
- ID menggunakan **UUID V4** (Lebih aman dari ID sekuensial 1, 2, 3).
